Process Scheduling

The process scheduling is the activity of the process manager that handles the removal of the running process from the CPU and the selection of another process on the basis of a particular strategy. Following are a few Process Scheduling algorithms:

  1. First-Come First Served
  2. Round Robin
  3. Shortest Job First
  4. Shortest Remaining Time First
  5. Priority and Multilevel Queue scheduling
First Come First Served

FCFS executes queued processes and requests by the order of their arrival. It is based on FIFO queue. It is rarely used in modern operating systems.

Go
Round Robin Scheduling

In RR scheduling, each process is assigned a fixed time slot in a cyclic way. It is one of the most commonly used technique in CPU scheduling as a core.

Go
Shortest Job First

SJF or SJN(shortest job next) selects the waiting process with the smallest execution time to execute next. It is optimal but practically infeasible scheduling algorithm.

Go
Shortest Remaining Time First

SRTF is preemptive version of SJF. In SRTF, the process with the smallest amount of time remaining until completion is selected to execute first.

Go
Priority Scheduling

This algorithm selects the process with the highest priority to be execute first. Same priority processes are executed on FCFS basis. It is one of the most common scheduling algorithms in batch systems.

Go
Multilevel Queue Scheduling

In MLQS there will be some number of ready queues with priority and their own scheduling algorithm to execute. For the process in a queue to execute, all the queues of priority higher than it should be empty.
